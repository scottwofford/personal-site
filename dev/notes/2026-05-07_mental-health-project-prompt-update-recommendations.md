### 3. The Type 3 → Type 6 security pattern explains a Scott→Maria dynamic the current prompt misses.

**Type 3 security direction** (from Scott_Enneagram.pdf): *"With most people, Threes make every effort to be diplomatic and well-mannered... But when Threes feel that their relationships are secure, they can be more open about expressing their anxieties and frustrations. They may keep a 'positive frame of mind' all day at work, only to come home and download their dissatisfaction onto their spouse or partner. ('I think my boss is going to go nuts on me when he finds out we still haven't got this report nailed down.') **Implication.** This is a *separate* Scott→Maria dynamic from the Type 7 → Type 1 anger one. Scott's day-end anxiety download onto Maria is Type 3 → Type 6 — Scott in Achiever mode all day, then dropping the burden on his partner because she's the safe space. The current prompt's "Scott suppresses his processing to stay supportive → feels constrained → resentment builds → stress triggers Type 1 anger" is one mechanism. The Type 3 → Type 6 mechanism is different: Scott's anxieties don't manifest as anger; they manifest as dropped-on-Maria worries. Maria experiences the difference, and the support she needs in each case is different.

Different presentations need different responses from Scott. The current prompt's single-mode framing means Scott's response template is one-size-fits-all when the stress mode determines what helps.

### 5b. Add a `<stress_mode_identification>` block to the prompt template

```
<stress_mode_identification>
When Scott describes a recent or upcoming charged interaction with Maria:
1. Identify which stress mode Scott is in (Type 1 anger? Type 9 autopilot? Type 6 anxiety-download? Or operating within healthy range?).
2. Identify which mode Maria is likely in (Type 8 confrontational? Type 5 cold-withdrawal? Type 4 moody-self-absorbed? Or operating within healthy range?).
3. Use the mode-pair to predict which dynamic is most likely active (see <key_facts>).
4. Match Scott's response to the mode-pair, not to a single Type-7/Type-2 frame.

Specifically: do NOT default to "Maria is in Type 2 and just needs validation" — that's only true some of the time. When she's in Type-8 mode, she needs direct engagement, not soft validation. When she's in Type-5 mode (cold withdrawal), she needs respected space, not pursuit.
</stress_mode_identification>
```
