A framework for tracking and communicating your current priorities.

How to Use This

Status Colors

  • Green: On track
  • Yellow: At risk, needs attention
  • Red: Blocked or behind

Updates Column

Brief status notes. Keep it scannable—save details for the tracker link.

Tracker Column

Link to the detailed tracking document (if one exists).

Best Practices

  1. Limit to 3-5 priorities - if everything is a priority, nothing is
  2. Update weekly - stale priorities docs lose credibility
  3. Share with your manager - alignment prevents surprises
  4. Include "what done looks like" - concrete outcomes, not activities
  5. Review quarterly - are these still the right priorities?

Template for Each Priority

### [Priority Name]

**Objective:** What are you trying to accomplish?

**Success Metrics:** How will you know when you're done?

**Current Status:** Green/Yellow/Red + brief explanation

**Key Milestones:**
- [ ] Milestone 1 (Date)
- [ ] Milestone 2 (Date)
- [ ] Milestone 3 (Date)

**Blockers/Risks:** What could derail this?

**Updates:**
- [Date]: Update here

Keep this simple. The goal is clarity and alignment, not documentation theater.
